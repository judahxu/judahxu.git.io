react-native和微信小程序

16年有时间就抽空看其它的知识(挺杂的)，主要是react-native，更加倾向于热更新方向。

  • https://zhuanlan.zhihu.com/p/23412169
  • http://www.lcode.org/react-native/
    Ruby环境 爬虫 Python、Node

brew - nvm - npm - node

  • 接触rn，得懂些HTML、CSS、JavaScript

    • http://www.liaoxuefeng.com/wiki/001434446689867b27157e896e74d51a89c25cc8b43bdb3000
    • http://www.runoob.com/

  • 搭建环境(对每个命令地址都要稍微记忆下，以后升级会有坑的)

    • http://reactnative.cn/翻译http://facebook.github.io/react-native/docs/getting-started.html
    • 编辑器 WebStorm(推荐) VisualStudioCode Atom
    • 调试工具 AndroidStudio Xcode

  • 可以在github从收藏热度最高或最低，或在最近更新里找到简单应用demo。（好的demo可以加快学习认知的进度）

    • https://github.com/poberwong/react-native-gank（清晰明了）
    • https://github.com/attentiveness/reading(代码结构规范)

  • 跟着demo或者一些网站讲解案例教程，从头敲一遍，遇到问题可以标注一下，先找度娘简单去理解，敲出来显示的效果会更加增进你的理解。完整的流程走下来会增加信心哟！

  • 多敲几个demo熟悉编辑器，熟悉语法，JS环境。中间遇到的各种问题就百度、google。对基础语法的加深理解。

  • 接下来可以看官方提供的example

    • https://github.com/facebook/react-native

  • 可以把自己的项目某一界面用rn来实现，增加实战经验。

总的来说还是要对HTML、CSS、JavaScript、React基础语法更加熟悉

对rn已经入门阶段，现在再看微信小程序，也是在这些基础语法之上。

  • https://mp.weixin.qq.com/debug/wxadoc/dev/
    官方源码看起来也没那么难懂了。
  • https://github.com/lypeer/wechat-weapp-gank（浅显易懂）
  • 小程序的小而美，刚开始确实让我们眼前一亮发现了新大陆，未来的App应该是这样子的吧。硬件上的提升带来软件上的飞跃。小程序的体验也还不错，慢慢等更新等体验更好。

http://gank.io/的API简单清晰易用，被大家用来做各种形式的app。这些demo也很值得我们去学习的

对于react-native的理解还有待完善。

有人说：只要能用js写出来app，更多人愿意用这种编码形式吧。
